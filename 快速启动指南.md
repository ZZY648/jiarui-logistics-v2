# 3 步启动系统

## 准备工作（只需做一次）

- 安装 Docker Desktop：https://www.docker.com/products/docker-desktop/
- 安装 Java 11+：https://adoptium.net/
- 安装 Maven：https://maven.apache.org/download.cgi

## 第 1 步：启动数据库和中间件

打开终端，进入项目 docker 目录，一条命令启动全部基础设施：

```bash
cd E:\研究内容\jiarui-logistics-v2\docker
docker compose up -d
```

等待 30 秒，然后验证：

```bash
docker ps
```

应该看到 4 个容器都在运行：jiarui-pg、jiarui-redis、jiarui-rabbitmq、jiarui-minio。

## 第 2 步：初始化数据库表

```bash
docker exec -i jiarui-pg psql -U jiarui -d jiarui_logistics < E:\研究内容\jiarui-logistics-v2\sql\001-schema.sql
```

验证表已建好：

```bash
docker exec -i jiarui-pg psql -U jiarui -d jiarui_logistics -c "\dt"
```

## 第 3 步：启动后端

```bash
cd E:\研究内容\jiarui-logistics-v2\backend
mvn spring-boot:run
```

看到 `Started JiaruiApplication` 就成功了。

## 这就完了，测试一下

新开一个终端：

```bash
curl -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{\"username\":\"admin\",\"password\":\"jiarui123\"}"
```

返回带 `accessToken` 的 JSON 就是登录成功。

## 常见问题

| 问题 | 解决 |
|------|------|
| docker 命令找不到 | 安装 Docker Desktop 后重启电脑 |
| 端口 5432 被占用 | 本机已有 PostgreSQL，先停掉：`net stop postgresql` |
| mvn 命令找不到 | 装 Maven 或直接用 IDE（IDEA 导入 pom.xml 运行） |
| 启动报数据库连接错 | 确认第 1 步的 `docker ps` 看到容器在运行 |
