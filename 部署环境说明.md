# 部署环境说明

> **部署类型必须选择 Node.js Web Service、Web Service 或 Docker。不要选择“Static Site / 静态网站”。**
>
> 首页文件位于 `server/public/index.html`，由 Express 后台在 `/` 路径返回。Node 项目根目录不需要单独放置 `index.html`。如果平台要求根目录必须有 `index.html`，说明当前选择的是纯静态托管，登录和业务接口无法运行。
项目现在支持从根目录直接构建和启动，不需要手动进入 `server` 目录。

## 标准 Node.js 托管

- Node.js：20、22 或 24
- 安装命令：`npm install`（有锁文件后可使用 `npm ci`）
- 启动命令：`npm start`
- 健康检查：`/health`

必须设置的生产环境变量：

- `NODE_ENV=production`
- `JWT_SECRET`：长随机字符串
- `SEED_PASSWORD`：首次初始化演示账号的密码

可选环境变量：

- `PORT`：平台通常会自动设置
- `DATA_FILE`：持久化数据文件绝对路径

生产环境默认不会读取本地的 `server/data.json`，而是使用 `server/.data/data.json`，避免把本地业务数据带到线上。

## Docker

```bash
docker build -t jiarui-logistics .
docker run -d --name jiarui-logistics -p 3000:3000 \
  -e JWT_SECRET=replace-with-random-secret \
  -e SEED_PASSWORD=replace-with-strong-password \
  -v jiarui-data:/app/data \
  jiarui-logistics
```

## 数据持久化

系统使用 JSON 文件保存数据。托管平台必须提供持久化磁盘，并将 `DATA_FILE` 指向该磁盘；否则平台重启或重新部署后，线上数据可能恢复为初始演示数据。
