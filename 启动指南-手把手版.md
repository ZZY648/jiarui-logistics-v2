# 启动指南 — 每个操作单独复制粘贴

## 前置检查：你装了 Docker 吗？

按键盘上的 `Win` 键（Windows 图标键），输入 `PowerShell`，点开。

把下面这行**完整复制**，在 PowerShell 窗口里**右键粘贴**，按**回车**：

```
docker --version
```

如果出来一串版本号（比如 `Docker version 24.0.7`）→ 你已安装 Docker，跳到下一步。
如果出来红色的报错 → 你没装 Docker，先去 https://www.docker.com/products/docker-desktop/ 下载安装。

---

## 前置检查：你装了 Java 吗？

复制粘贴这一行，回车：

```
java -version
```

如果出来版本号 → 有 Java。
如果报错 → 去 https://adoptium.net/ 下载安装（选 Temurin 17）。

---

## 正式开始：共 4 条命令

先确认你的终端**当前在项目目录**。复制粘贴这行，回车：

```
cd E:\研究内容\jiarui-logistics-v2
```

> 这句在做什么：把终端的工作目录切换到你的项目文件夹。

---

### 命令 ① 启动数据库

```
docker compose -f docker/docker-compose.yml up -d
```

> 这句在做什么：用 Docker 在后台启动 PostgreSQL（存数据的）、Redis（缓存）、RabbitMQ（消息）、MinIO（存图片的），一共 4 个东西。`-d` 表示后台运行，不占窗口。

复制粘贴，回车。等它跑完（大概 30 秒，窗口里会看到各种下载/启动的日志）。

---

### 命令 ② 创建数据库表

```
docker exec -i jiarui-pg psql -U jiarui -d jiarui_logistics < sql/001-schema.sql
```

> 这句在做什么：把项目里写好的 22 张数据库表，塞进刚才启动的 PostgreSQL 里面。"jiarui-pg" 是数据库容器的名字。

复制粘贴，回车。如果成功，你会看到很多 `CREATE TABLE` 的输出。

验证一下（可选）：

```
docker exec -i jiarui-pg psql -U jiarui -d jiarui_logistics -c "\dt"
```

应该看到 20+ 张表的名字。

---

### 命令 ③ 编译项目

```
cd backend
```

> 切进后端代码目录。

然后：

```
mvnw.cmd clean package -DskipTests
```

> 这句在做什么：用 Maven 编译 Java 项目。`-DskipTests` 是跳过单元测试（先跑起来再说）。**第一次运行会下载很多依赖包，可能需要 3-5 分钟**，耐心等。

如果 `mvnw.cmd` 不识别（说明项目里没有 Maven Wrapper），换这个：

```
mvn clean package -DskipTests
```

---

### 命令 ④ 启动系统

```
mvnw.cmd spring-boot:run
```

或：

```
mvn spring-boot:run
```

> 这句在做什么：启动你的物流管理系统后端。

等看到 `Started JiaruiApplication in X seconds` 就成功了。**这个窗口不要关**，它就是正在运行的系统。

## 测试登录

另外开一个 PowerShell 窗口，复制粘贴这一整行，回车：

```
curl -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{\"username\":\"admin\",\"password\":\"jiarui123\"}"
```

如果回来一串 JSON，里面有 `accessToken` 字样，登录成功。

## 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| docker compose 报 `not found` | Docker Desktop 没装或没启动 | 从开始菜单打开 Docker Desktop，等它变绿 |
| 端口 5432 被占用 | 你电脑上已经有 PostgreSQL | `net stop postgresql` 先停掉旧的 |
| mvn 报 `not found` | 没装 Maven | 装一个或用 IDE 打开项目运行 |
| 启动报数据库连接失败 | 命令①没成功 | 重新跑命令①，确保 4 个容器在运行 |
| curl 命令不识别 | Windows 不自带 curl | 用浏览器访问 http://localhost:8080/actuator/health |
