# 佳瑞物流管理平台使用说明（当前版本）

> 适用版本：佳瑞物流管理系统 V2.0 Node.js 版  
> 文档更新日期：2026 年 7 月 13 日  
> 系统默认地址：`http://127.0.0.1:3000/`

---

## 一、系统概述

佳瑞物流管理平台用于管理物流运输业务的完整过程，当前已覆盖：

- 数据看板
- 客户与地址簿管理
- 运单创建、查询和状态流转
- 车辆推荐、司机调度和派车
- 司机端运输任务及 GPS 定位上报
- 运单轨迹追踪
- 费用录入、审核和成本核算
- 月度对账单生成、发送和确认
- 车辆、司机和系统用户管理
- 客户端运单、轨迹及对账单查询

当前版本采用本地 Node.js 服务运行，业务数据保存在 `server/data.json`。适合内部测试、业务演示和小规模使用。多人高并发或正式生产运营前，建议迁移至 PostgreSQL 等正式数据库。

## 二、启动与访问

### 2.1 一键启动

在项目根目录双击：

```text
启动系统.bat
```

启动成功后，浏览器访问：

```text
http://127.0.0.1:3000/
```

关闭启动窗口会停止系统服务。

### 2.2 命令行启动

打开 PowerShell，执行：

```powershell
cd E:\研究内容\jiarui-logistics-v2\server
npm install
npm start
```

| 使用端 | 地址 | 适用人员 |
|---|---|---|
| 管理后台 | `http://127.0.0.1:3000/` | 管理员、运营、调度、财务、客服、老板 |

## 四、角色权限说明

### 4.1 管理员

管理员拥有全部业务权限，包括：

- 查看数据看板
- 创建、流转和删除运单
- 调度车辆和司机
- 录入、审核和删除费用
- 生成、发送和确认对账单
- 管理客户、地址、车辆、司机和用户
- 查看全部轨迹

### 4.2 运营主管

可管理运单、调度、费用、客户和车队，但不能管理系统用户，也不能生成财务对账单。

### 4.3 调度员

主要负责：

- 新建运单
- 查看运单
- 推荐车辆和派车
- 推进运输状态
- 查看轨迹
- 录入运输费用

### 4.4 财务人员

主要负责：

- 查看运单
- 录入并审核费用
- 查看成本核算
- 维护客户资料
- 生成、发送和确认对账单

### 4.5 客服与管理层

客服可查看运单、客户和轨迹；老板账号主要查看经营看板和运输轨迹。

### 4.6 客户账号

客户只能查看与自己客户档案关联的数据：

- 本客户运单
- 本客户运输轨迹
- 本客户对账单
- 本客户看板数据

客户无法查看其他客户、车辆司机资源或内部调度数据。

### 4.7 司机账号

司机只能：

- 查看分配给本人的在途运单
- 查看提货和送货地址
- 开启或停止 GPS 定位
- 向本人承运的运单上报位置

## 五、标准业务流程

系统推荐按以下顺序完成一票运输业务：

```text
维护客户 → 创建运单 → 调度派车 → 确认装车 → 确认发车
→ 运输中 → 确认到达 → 确认签收 → 完成归档
→ 录入/审核费用 → 核算成本 → 生成对账单 → 发送 → 确认
```

请不要跳过业务状态。系统会阻止不符合规则的状态变更。

## 六、客户管理

入口：左侧菜单“客户管理”。

### 6.1 新增客户

1. 点击“新增客户”。
2. 填写公司名称、简称、联系人、联系电话和地址。
3. 选择结算方式。
4. 填写信用额度和折扣率。
5. 保存客户。

系统会自动生成类似 `KH001` 的客户编码。

### 6.2 地址簿

每个客户可以维护多个提货或收货地址。创建运单时可以根据客户信息填写运输站点。

### 6.3 停用和删除

- “停用”保留历史数据，但客户不再作为正常客户使用。
- 彻底删除仅允许没有关联运单的客户。
- 有历史运单的客户不能彻底删除，以保证业务记录完整。

## 七、创建运单

入口：左侧菜单“运单管理” → “新建运单”。

### 7.1 必填内容

- 客户
- 货物名称
- 货物重量，且必须大于 0
- 至少一个提货站点
- 至少一个送货站点

可继续填写货物类型、体积、件数、时效和备注。

### 7.2 提货与送货信息

每个站点包含：

- 联系人和电话
- 省、市、区县
- 详细地址
- 可选的高德经纬度

如果已经从高德地图获得精确坐标，建议填写经纬度。系统会将其作为地图起点和终点。

如果不填写经纬度，系统会使用城市或区县中心点估算，并在地图中标注“区域估算位置”。区域估算不能等同于门牌号精确位置。

### 7.3 运单编号和报价

系统自动生成形如：

```text
YD202607130001
```

的运单编号，并根据件数和重量生成当前演示报价。

新建运单成功后的初始状态为“已确认”，可以进入调度流程。

## 八、运单状态流转

正常状态顺序如下：

| 当前状态 | 可进入状态 | 说明 |
|---|---|---|
| 草稿 | 已确认、已取消 | 草稿确认或取消 |
| 已确认 | 已调度、已取消 | 等待派车 |
| 已调度 | 已装车、已取消、异常 | 已分配车辆和司机 |
| 已装车 | 运输中、异常 | 货物已完成装车 |
| 运输中 | 已到达、异常 | 司机正在运输 |
| 已到达 | 已签收、异常 | 到达送货地 |
| 已签收 | 已完成 | 完成业务归档 |
| 异常 | 重新调度、恢复装车、恢复运输、取消 | 根据实际情况处理 |

业务规则：

- 派车必须同时选择车辆和司机。
- 车辆必须处于空闲状态。
- 司机必须处于可用状态。
- 车辆额定载重必须大于或等于货物重量。
- 完成或取消后，系统自动释放车辆和司机。
- 确认装车后，提货站点变为完成。
- 确认到达后，送货站点变为已到达。
- 确认签收后，系统生成签收记录，并将送货站点变为完成。

只能删除“草稿”或“已取消”状态的运单。存在费用记录的运单不能删除。

## 九、调度派车

入口：左侧菜单“调度派车”。

### 9.1 推荐车辆

系统根据货物重量和车辆类型，从空闲且载重满足要求的车辆中推荐车辆。

车辆推荐只是辅助决策，调度人员仍需核对：

- 车辆载重和容积
- 车辆状态
- 司机可用状态
- 线路和车型要求

### 9.2 正式派车

1. 在待调度运单中选择车辆。
2. 选择司机。
3. 点击“派车”。
4. 派车成功后，运单进入“已调度”。
5. 对应车辆变为运输占用，司机变为执行任务。

系统禁止一台车辆或一名司机同时执行多张活动运单。

## 十、司机端使用

访问：

```text
http://127.0.0.1:3000/driver/
```

使用 `driver1`～`driver6` 登录。

### 10.1 查看运输任务

司机端只显示分配给当前司机，且状态为以下类型的运单：

- 已装车
- 运输中
- 已到达

点击运单可以查看货物、客户、提货地址和送货地址。

### 10.2 开启 GPS 定位

只有“已装车”或“运输中”的运单可以上报定位。

1. 打开运单详情。
2. 点击“开始 GPS 定位”。
3. 浏览器询问位置权限时选择“允许”。
4. 保持司机端页面打开。
5. 页面显示上报次数和最新坐标。

浏览器提供的是 WGS-84 坐标，系统会自动转换为高德地图使用的 GCJ-02 坐标，避免地图偏移。

### 10.3 停止定位

点击“停止定位”即可结束本次定位。页面会显示本次上报次数，并保留当前运单详情。

返回运单列表或退出登录时，系统会静默停止定位，不会产生重复提示。

### 10.4 定位失败处理

- 提示“定位权限被拒绝”：在浏览器网站权限中允许位置访问。
- 长时间无法定位：到室外或信号较好的位置重试。
- 电脑浏览器测试时，位置可能来自系统或网络估算，不一定等同于车辆真实 GPS。
- 手机需开启系统定位功能，并允许浏览器使用精确位置。

## 十一、轨迹追踪

入口：左侧菜单“轨迹追踪”。

输入运单号并查询后，地图显示：

- 蓝色虚线：计划路线
- 绿色实线：司机设备实际上报轨迹
- “取”标记：提货站点
- “达”标记：送货站点
- 绿色当前位置标记：司机最新上报位置

计划路线的首尾点始终来自该运单的提货和送货站点。

### 11.1 坐标准确度

- 建单时填写高德经纬度：地图使用精确坐标。
- 未填写经纬度：使用区县或城市中心估算。
- 地图弹窗出现“区域估算位置”时，说明该点不是详细门牌坐标。

### 11.2 地图加载较慢

Leaflet 和 ECharts 已改为本地资源，因此页面组件可快速加载。但地图底图瓦片仍需要访问高德地图网络服务。

网络较慢时：

- 计划路线、实际轨迹和站点标记仍会先显示。
- 底图可能稍后出现。
- 页面会提示“底图网络较慢，路线和站点仍可正常查看”。

如果内网无法访问高德瓦片，需要为系统部署内网地图瓦片服务。

## 十二、费用与成本核算

入口：运单详情中的“记录费用”，或“费用对账”页面。

### 12.1 费用类型

当前支持：

- 油费
- 过路费
- 司机提成
- 装卸费
- 罚款
- 其他费用

金额必须大于 0。

### 12.2 费用审核

- 财务和财务主管录入的费用会直接进入已审核状态。
- 调度或运营录入的费用为待审核状态。
- 管理员、财务主管和财务可以点击“审核”。
- 只有已审核费用计入成本快照。

### 12.3 成本快照

点击“核算成本”后，系统计算：

- 直接成本
- 车辆折旧和保险分摊
- 总成本
- 利润
- 利润率

运单进入对账流程后，费用不能再删除。

## 十三、对账管理

入口：左侧菜单“费用对账”。

### 13.1 生成对账单

1. 选择客户。
2. 选择年份和月份。
3. 点击生成对账单。

系统只纳入：

- 状态为“已完成”的运单
- 结算状态为“待入账”的运单
- 完结日期位于所选账期内的运单

### 13.2 对账单状态

```text
草稿 → 已发送 → 已确认
```

- 草稿可以删除。
- 草稿不能直接确认。
- 只有已发送的对账单可以确认。
- 已确认后，对应运单结算状态同步为已确认。
- 已发送或已确认的对账单不能删除。

客户可以在客户端查看属于自己的对账单和明细。

## 十八、常见问题

### 18.1 页面打不开

确认启动窗口仍在运行，并访问：

```text
http://127.0.0.1:3000/
```

### 18.2 端口被占用

检查是否已经启动过一份系统。可以关闭旧的 Node.js 服务后重新启动。

### 18.3 登录失败

- 检查用户名和密码。
- 检查账号是否被禁用。
- 默认密码为 `jiarui123`。

### 18.4 无法派车

可能原因：

- 运单不是“已确认”或可重新调度状态。
- 没有同时选择车辆和司机。
- 车辆不空闲。
- 司机不在岗。
- 车辆载重不足。
- 车辆或司机正在执行其他活动运单。

### 18.5 GPS 没有位置

- 检查运单是否为已装车或运输中。
- 检查是否使用了该运单分配的司机账号。
- 检查手机定位和浏览器位置权限。
- 保持司机端页面打开。

### 18.6 地图起终点不够精确

说明建单时没有填写精确高德经纬度，系统使用了区域估算。请在新建运单时填写真实经纬度，或后续接入高德地理编码服务。

### 18.7 地图底图加载慢

通常为高德地图瓦片网络访问较慢。路线和站点不依赖底图，可以先查看；正式内网部署建议配置内网瓦片服务。

### 18.8 无法生成对账单

确认：

- 所选客户存在已完成运单。
- 运单完结日期在所选月份。
- 运单尚未进入其他对账单。

## 十九、当前版本使用边界

当前版本已经具备完整演示业务闭环，但仍需注意：

- 使用 JSON 文件存储，不适合大量并发写入。
- 地图详细地址自动解析尚未配置商业地图 API Key。
- 计划路线当前根据运单站点生成，不代表实时道路导航路线。
- 浏览器 GPS 依赖设备和浏览器权限。
- 没有短信、微信、电子签章和正式发票接口。
- 没有企业级数据库事务、审计日志和自动灾备。

正式生产上线前建议完成数据库迁移、HTTPS、环境变量密钥、操作审计、自动备份和地图服务配置。

## 二十、推荐日常操作顺序

### 调度人员

```text
检查客户资料 → 新建运单 → 核对重量和地址 → 推荐车辆
→ 选择车辆和司机 → 派车 → 按实际节点推进状态 → 查看轨迹
```

### 司机

```text
登录司机端 → 查看本人任务 → 核对提送货地址
→ 装车/发车后开启定位 → 到达后停止定位 → 配合签收
```

### 财务人员

```text
检查费用 → 审核费用 → 核算成本 → 检查已完成运单
→ 生成月度对账单 → 发送客户 → 确认对账
```

### 管理员

```text
维护账号权限 → 检查基础数据 → 查看运营看板
→ 定期备份 data.json → 检查启动日志中的一致性警告
```
